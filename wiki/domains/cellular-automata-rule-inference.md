# Cellular Automata Rule Inference

## Overview
Given a trajectory of a cellular automaton (sequence of grid states), infer the transition rule and predict future states. This is directly inspired by the NCA pre-pre-training paper (Lee et al., 2025) — the key finding was that forcing models to do pure rule inference from CA trajectories develops transferable attention mechanisms. As an RLVR environment, we make this explicit: the agent sees a trajectory and must predict the next state. Verification is trivial — simulate the actual CA.

The deep insight: every CA sequence is generated by a single latent rule. With no semantic shortcuts, the model must develop pure **latent rule inference** — the same computational primitive that underlies language modeling, code understanding, and scientific reasoning.

## Verification Mechanism
```python
def verify(task_type: str, trajectory: list[np.array], rule: CARule, answer: Any) -> float:
    if task_type == "predict_next":
        # Given trajectory t0..tN, predict tN+1
        actual_next = rule.step(trajectory[-1])
        return 1.0 if np.array_equal(answer, actual_next) else 0.0
    
    elif task_type == "predict_k_ahead":
        # Predict k steps ahead
        state = trajectory[-1]
        for _ in range(answer["k"]):
            state = rule.step(state)
        return 1.0 if np.array_equal(answer["prediction"], state) else 0.0
    
    elif task_type == "identify_rule":
        # Given trajectory, identify the rule (as a lookup table or network weights)
        # Verify by checking if the identified rule reproduces the trajectory
        for i in range(len(trajectory) - 1):
            predicted = answer["rule"].step(trajectory[i])
            if not np.array_equal(predicted, trajectory[i + 1]):
                return 0.0
        # Also check on held-out steps
        state = trajectory[-1]
        for _ in range(10):
            predicted = answer["rule"].step(state)
            actual = rule.step(state)
            if not np.array_equal(predicted, actual):
                return 0.0
            state = actual
        return 1.0
    
    elif task_type == "classify_dynamics":
        # Classify as: fixed point, periodic, chaotic, complex
        actual_class = classify_ca_dynamics(trajectory, rule)
        return 1.0 if answer == actual_class else 0.0
```

## Dataset Sources
- **Procedural generation**: Unlimited. The paper's method: randomly initialize neural network parameters θ for the CA transition function, sample initial states uniformly, filter by gzip compressibility (>50% ratio for non-trivial dynamics).
- **Wolfram elementary CA**: 256 rules, extensively studied.
- **Life-like CA**: B/S notation rules (e.g., B3/S23 = Game of Life).
- **NCA from the paper**: 2D grids (12×12), n=10 states, 3×3 conv + MLP transition.
- **Lenia**: Continuous CA with smooth kernels — richer dynamics.
- **Multi-state CA**: Higher alphabet sizes increase complexity.

## Task Format
- **Input**: Sequence of grid states: "t0: [[0,1,2],[3,0,1],[2,1,0]], t1: [[1,2,0],[0,1,2],[1,0,1]], t2: ?"
- **Output**: Predicted grid state for t2

## Difficulty Curriculum
- Level 1: Elementary 1D CA (small grid, 2 states, 3-cell neighborhood)
- Level 3: 2D CA with small grids (8×8, 2-4 states)
- Level 5: 2D NCA with 10 states on 12×12 grids (paper's setting)
- Level 7: Large grids (32×32+), complex rules, long prediction horizons
- Level 9: Infer rules from partial/noisy observations, predict far-future states
- Level 10: Identify whether two different-looking trajectories come from the same rule class

## Limitations & Risks
- At high complexity, CA dynamics can be computationally irreducible (Wolfram's principle) — prediction becomes as hard as simulation. This is a feature, not a bug: it forces the model to learn the rule rather than take shortcuts.
- Gzip filtering is crucial to avoid trivially predictable (fixed point) or totally random (chaotic) sequences.
