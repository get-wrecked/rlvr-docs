# Latent Concept Learning (Few-Shot Rule Inference)

## Overview
Given a few examples generated by a hidden concept/rule, identify the concept and apply it to new instances. This is the meta-learning formulation: each "task" is defined by a latent concept, and the agent must do in-context learning to identify and apply it.

This directly implements the mechanism the NCA paper identifies as the source of transfer: "Each sequence is generated by a single latent rule that the model must infer from context and then apply consistently." We generalize beyond CA to ANY latent rule system.

## Verification Mechanism
```python
def verify(concept: LatentConcept, test_instances: list, answer: Any) -> float:
    if isinstance(concept, ClassificationConcept):
        # Agent classifies new instances according to the hidden concept
        correct = 0
        for instance, prediction in zip(test_instances, answer["predictions"]):
            if prediction == concept.classify(instance):
                correct += 1
        return correct / len(test_instances)
    
    elif isinstance(concept, TransformationConcept):
        # Agent applies the hidden transformation to new inputs
        correct = 0
        for instance, prediction in zip(test_instances, answer["outputs"]):
            if prediction == concept.transform(instance):
                correct += 1
        return correct / len(test_instances)
    
    elif isinstance(concept, SequenceConcept):
        # Agent continues a sequence governed by a hidden rule
        actual_continuation = concept.generate(len(answer["continuation"]))
        matches = sum(1 for a, b in zip(answer["continuation"], actual_continuation) if a == b)
        return matches / len(actual_continuation)
```

## Dataset Sources
- **Procedural generation**: Unlimited. Generate latent concepts from:
  - **Arithmetic rules**: "add 3", "multiply by input mod 7", "Fibonacci-like recurrence"
  - **String transformations**: "reverse", "sort characters", "remove vowels", "Caesar shift by K"
  - **Logical rules**: "XOR of first two features", "majority vote", "parity"
  - **Geometric concepts**: "all convex shapes", "shapes with exactly 2 holes"
  - **Relational concepts**: "A is always before B", "count of X equals count of Y"
- **Omniglot/Mini-ImageNet**: Few-shot image classification (visual concepts)
- **SCAN**: Compositional generalization benchmark
- **gSCAN**: Grounded compositional generalization
- **Meta-learning benchmarks**: Harmonics, Taskonomy

## Task Format
- **Input**: "Examples of concept: 'apple'→True, 'banana'→True, 'car'→False, 'grape'→True, 'table'→False. Classify: 'cherry'→?"
- **Output**: "True" (concept = fruits)

## Difficulty Curriculum
- Level 1: Simple attribute-based concepts (color, size, parity)
- Level 3: Compositional concepts (red AND large, or XOR of features)
- Level 5: Relational concepts (A > B, counts equal)
- Level 7: Abstract/structural concepts (symmetry, recursion)
- Level 9: Multi-step inference (concept requires chaining multiple sub-rules)
- Level 10: Concepts that require inventing new abstractions
