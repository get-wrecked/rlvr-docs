# Synthetic Grammar Induction

## Overview
Given sequences generated by a hidden formal grammar (Dyck languages, PCFGs, HMMs, L-systems), infer the grammar and predict continuations. The NCA paper compared against K-Dyck pre-pre-training and found NCA superior, but Dyck itself still beat scratch. This domain encompasses the broader class: any synthetic sequence generated by a hidden rule system.

The key value: these environments isolate **pure structural reasoning** from semantic content. The model must learn to track hierarchical structure, long-range dependencies, and latent state — the same capabilities needed for code (bracket matching), natural language (syntax), and mathematics (proof structure).

## Verification Mechanism
```python
def verify(task_type: str, grammar: Grammar, sequence: list, answer: Any) -> float:
    if task_type == "predict_next":
        # Given prefix, predict valid next token(s)
        valid_continuations = grammar.valid_next_tokens(sequence)
        return 1.0 if answer in valid_continuations else 0.0
    
    elif task_type == "complete_sequence":
        # Complete a partial sequence validly
        completed = sequence + answer["completion"]
        return 1.0 if grammar.accepts(completed) else 0.0
    
    elif task_type == "classify_membership":
        # Is this sequence in the language?
        actual = grammar.accepts(sequence)
        return 1.0 if answer == actual else 0.0
    
    elif task_type == "infer_grammar":
        # Given positive/negative examples, infer the grammar
        for pos in grammar.positive_examples:
            if not answer["grammar"].accepts(pos):
                return 0.0
        for neg in grammar.negative_examples:
            if answer["grammar"].accepts(neg):
                return 0.0
        return 1.0
    
    elif task_type == "hidden_state_tracking":
        # Given an HMM's emissions, infer hidden state sequence
        # Viterbi decoding gives the correct answer
        correct = viterbi(grammar.hmm, sequence)
        matches = sum(1 for a, c in zip(answer, correct) if a == c)
        return matches / len(correct)
```

## Dataset Sources
- **Procedural generation**: Unlimited.
  - **Dyck languages**: K-Dyck with varying K (bracket types) and depth limits
  - **PCFGs**: Random probabilistic context-free grammars
  - **Regular languages**: Random DFA/NFA, generate accepted/rejected strings
  - **L-systems**: Lindenmayer systems with varying complexity
  - **HMMs**: Random Hidden Markov Models with varying hidden states
  - **Markov chains**: Various orders, transition matrix complexities
- **Formal language theory benchmarks**: Standard examples from Sipser, Hopcroft
- **The NCA paper's Dyck baseline**: K=128 Dyck language, infinite depth

## Task Format
- **Input**: "Sequence: ( ( ) [ ] ( [ ) — is this a valid Dyck-2 string? If not, what should replace the last token?"
- **Output**: "Invalid. Replace ']' with ')' to get a valid prefix."

## Difficulty Curriculum
- Level 1: Dyck-1 (single bracket type), shallow depth
- Level 3: Dyck-K with K=4-8, moderate depth
- Level 5: PCFGs with 20+ rules, ambiguous grammars
- Level 7: L-systems with complex rewriting rules
- Level 9: Infer hidden grammar from examples only (no grammar specification given)
- Level 10: Distinguish subtly different grammars from their sample distributions

## Limitations & Risks
- Formal grammar tasks can be "too clean" — real language has noise, exceptions, ambiguity. Balance with natural language tasks.
- Very deep nesting can exceed model context. Control depth in curriculum.
- This is a Stage 1 (pre-training) environment — develops raw structural reasoning that transfers to everything else.
