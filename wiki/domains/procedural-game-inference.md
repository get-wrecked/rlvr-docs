# Procedural Game Rule Inference

## Overview
Given traces of gameplay from a novel, procedurally generated game (board state transitions, legal/illegal moves, win/loss outcomes), infer the rules and then play optimally or predict outcomes. This is the "meta" version of game playing: instead of learning to play ONE game, learn to recognize and play ANY game from observation.

This is the purest form of "recognize the rules of the game" from the user's intelligence theory. Every trace is generated by hidden rules. The model must inverse-engineer those rules, then apply them.

## Verification Mechanism
```python
def verify(task_type: str, game: ProceduralGame, answer: Any) -> float:
    if task_type == "predict_legal_moves":
        # Given game state, predict which moves are legal
        actual_legal = set(game.legal_moves(answer["state"]))
        predicted_legal = set(answer["predicted_legal"])
        return set_f1(predicted_legal, actual_legal)
    
    elif task_type == "predict_outcome":
        # Given a complete game trace, predict winner
        actual_winner = game.evaluate_winner(answer["trace"])
        return 1.0 if answer["predicted_winner"] == actual_winner else 0.0
    
    elif task_type == "play_move":
        # Given state, play a legal move (any legal move gets partial credit, good move gets full)
        if answer["move"] not in game.legal_moves(answer["state"]):
            return 0.0
        # Evaluate move quality
        quality = game.evaluate_move(answer["state"], answer["move"])
        return quality
    
    elif task_type == "describe_rules":
        # Given traces, output rules in structured format — verify by simulating
        for test_trace in game.test_traces:
            if not answer["rules"].validates(test_trace):
                return 0.0
        return 1.0
```

## Dataset Sources
- **Procedural generation**: Unlimited.
  - Generate random board game rules: grid sizes, piece types, movement rules, capture rules, win conditions
  - Generate random card game rules: hand sizes, play rules, scoring
  - Generate random tile-placement rules
  - OpenSpiel framework can define custom games programmatically
- **Zillions of Games**: 2000+ user-created abstract strategy games (rules + game records)
- **BoardGameGeek**: Game rule descriptions (for NL→formal rule extraction)
- **General Game Playing (GGP) competition**: Games described in GDL (Game Description Language)

## Task Format
- **Input**: "Here are 5 game traces from an unknown board game on a 5x5 grid: [traces]. In this new position [state], what moves are legal?"
- **Output**: List of legal moves

## Difficulty Curriculum
- Level 1: Simple movement rules (one piece type, adjacent moves)
- Level 3: Multiple piece types, capture mechanics
- Level 5: Complex win conditions (territory, connection, elimination)
- Level 7: Games with hidden information or stochastic elements
- Level 9: Infer rules AND play optimally in a novel game from few traces
- Level 10: Identify rule changes mid-game (the rules themselves evolve)

## Limitations & Risks
- Complex games may require many traces to infer rules reliably. Control trace count in difficulty.
- Playing optimally requires both rule inference AND strategic reasoning — very challenging.
- Start with rule inference only, add optimal play as difficulty increases.
