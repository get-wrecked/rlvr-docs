# Anagram and Wordplay

## Overview

This domain covers word-level puzzles: solving anagrams, answering crossword clues, solving word riddles, and other word games that require vocabulary knowledge and constraint satisfaction. The model must manipulate letters, recognize word patterns, and apply lexical knowledge.

Verification is exact and elegant: check if the answer is a valid word (dictionary lookup) that satisfies the puzzle constraints (correct letter set for anagrams, correct length and crossing letters for crosswords). This domain is unusual in that verification is often cheaper than solving — checking that "LISTEN" is an anagram of "SILENT" is trivial, but finding the anagram is hard.

## Verification Mechanism

**Verification type: Exact**

### Anagram verification
```python
def verify_anagram(prediction: str, source: str) -> int:
    pred_sorted = sorted(prediction.lower().replace(" ", ""))
    source_sorted = sorted(source.lower().replace(" ", ""))
    is_valid_word = prediction.lower() in dictionary
    is_anagram = pred_sorted == source_sorted
    return 1 if is_valid_word and is_anagram else 0
```

### Crossword clue verification
```python
def verify_crossword(prediction: str, clue: str, length: int, 
                     pattern: str = None, gold_answer: str = None) -> int:
    if len(prediction) != length:
        return 0
    if pattern and not matches_pattern(prediction, pattern):
        return 0  # e.g., pattern "S_I_N_E" constrains certain positions
    if gold_answer:
        return 1 if normalize(prediction) == normalize(gold_answer) else 0
    return 0  # Without gold answer, cannot verify semantics
```

### Word puzzle verification
```python
def verify_word_puzzle(prediction: str, constraints: dict) -> int:
    word = prediction.lower()
    if not word in dictionary:
        return 0
    if 'length' in constraints and len(word) != constraints['length']:
        return 0
    if 'contains' in constraints and constraints['contains'] not in word:
        return 0
    if 'starts_with' in constraints and not word.startswith(constraints['starts_with']):
        return 0
    if 'letters' in constraints:
        if sorted(word) != sorted(constraints['letters']):
            return 0
    return 1
```

All verification is fully programmatic, deterministic, and fast. Dictionary lookup is O(1) with a hash set. Constraint checking is O(n) in word length.

**Scalability**: High through generation. Anagrams can be generated from any dictionary (take a word, shuffle its letters). Crossword clue-answer pairs number in the millions from newspaper archives. Word constraint puzzles can be generated by starting with a word and creating constraints it satisfies. The generation pipeline is:
1. Pick a word from the dictionary
2. Generate a puzzle (shuffle letters, create clue, set constraints)
3. Gold answer = the original word

## Difficulty Curriculum

1. **Level 1 — Short anagrams**: 4-5 letter anagram with a single valid solution. "TACS" -> "CATS".
2. **Level 2 — Longer anagrams**: 7-8 letter anagrams. Multiple valid rearrangements may exist, increasing search space.
3. **Level 3 — Standard crossword clues**: Direct definition clues. "Capital of France (5)" -> "PARIS".
4. **Level 4 — Multi-word anagrams**: Rearrange letters into a phrase. "ASTRONOMERS" -> "MOON STARERS".
5. **Level 5 — Cryptic crossword clues**: Clues with wordplay, double definitions, hidden words, and containers. "Flower found in Thames?" -> "ASTER" (hidden in thAMESter... actually "flower" = river = Thames). Cryptic clues are notoriously difficult.
6. **Level 6 — Constrained word generation**: Find a word satisfying multiple simultaneous constraints (length, letter positions, contained substrings, excluded letters).
7. **Level 7 — Complex wordplay**: Spoonerisms, portmanteaus, palindrome construction, and multi-step wordplay puzzles requiring creative linguistic manipulation.

## Limitations & Risks

- **Multiple valid answers**: An anagram of "LISTEN" could be "SILENT," "ENLIST," or "TINSEL." Without specifying which is wanted, verification must accept any valid answer — but crossword verification typically requires one specific answer.
- **Dictionary dependence**: The validity check depends on the dictionary used. Proper nouns, archaic words, and slang may or may not be included.
- **Semantic clue verification**: For crossword clues, verifying that the answer correctly matches the clue's meaning (not just length and pattern) requires the gold answer. Without it, only format constraints can be checked.
- **LLM weakness at letter manipulation**: Current LLMs are notoriously poor at character-level operations (counting letters, rearranging letters). Tokenization obscures individual characters. This makes anagram solving genuinely difficult for LLMs, which could mean sparse rewards in RLVR.
- **Cultural knowledge**: Crossword clues often rely on cultural references, abbreviations, and wordplay conventions specific to certain traditions (British cryptic vs. American standard crosswords).
- **Cheating via memorization**: Models that have seen crossword datasets during pretraining may memorize clue-answer pairs rather than learning to solve new puzzles.
